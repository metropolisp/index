Philosophy

Communal Computing

Computing Shouldn't Be Single-Player. Vouch For Your Friends, Lets Them Join Your Machine And Get With The Program(ming).

Spaced Out

Humans Operate In More That 1.5 Dimensions; So Should The Ways We Interact With Interfaces. The system state lives on the server — your browser is merely projection.

Live & Reflective

Every session is mutable. Every object inspectable. Every function redefinable in motion.

Homoiconoclasm

Code‑Should‑Be‑Data & Data‑Should‑Be‑Code Whenever Possible.

Applications Not Appliances

Everything function should be able to be called / applied to whatever you want -- not just what one explicitly makes available.

FAQs

Q: What is Ozu?

A: Ozu Is A Fork Of The Great CLOG Project. Instead Of Being A General-Purpose Graphical User-Interface Toolkit Like CLOG, Ozu Leans Into The Interactiveness Of The System Plus The Portability Of It Being Browser-Bound In An Attempt To Make A General Purpose Computing Environment You Can Control "From Anywhere".

Q: How does it work?

A: The Big Idea In Ozu is `Over-The-Wire` (In-fact Ozu is a forced acryonym meaning that); Notably We Run A Server And We Talk To It Via Our Client. So Any Browser / User With Access Has Access To A Real Lisp Image And Any Depends On The Host System. We Reduce the Amount Of Client Code Being Ran.
